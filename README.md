# 日報管理システム (Daily Report System)

## 概要
本プロジェクトは、JavaおよびSpring Bootを用いて開発した「日報管理システム」です。
従業員が日報を登録・閲覧できるWebアプリケーションであり、実際の開発業務の流れ（画面設計書、ER図、テーブル定義書などのドキュメントに基づく実装）を想定して作成しました。

管理者と一般従業員の権限に応じたアクセス制御（表示・操作制限）を実装しており、本格的なWebアプリケーションの要件を満たす構成となっています。

## 使用技術
* **バックエンド**: Java / Spring Boot / Spring Data JPA / Spring Security
* **フロントエンド**: HTML / CSS / Thymeleaf / Bootstrap
* **データベース**: MySQL
* **その他**: DBViewer, Docker（デプロイ用環境として想定）

## 機能一覧
本システムは、以下の主な機能を備えています。

### 1. 認証・認可（ログイン）機能
* 従業員の社員番号とパスワードを使用したログイン機能
* 権限（一般 / 管理者）に応じたアクセス制御
  * **管理者**: 全従業員の情報管理、および全従業員の日報の閲覧・管理が可能
  * **一般**: 自身が登録した日報のみ閲覧・操作が可能

### 2. 従業員管理機能（※管理者のみアクセス可能）
* **従業員一覧**: 登録されている従業員の一覧表示
* **従業員詳細**: 従業員の詳細情報の確認
* **従業員新規登録**: 新規従業員の登録（バリデーションチェック含む）
* **従業員更新**: 既存従業員情報の編集
* **従業員削除**: 従業員情報の論理削除（※紐づく日報情報も同時に論理削除）

### 3. 日報管理機能
* **日報一覧**: 日報の一覧表示（一般権限の場合は自身の日報のみ表示）
* **日報詳細**: 日報の詳細情報の確認
* **日報新規登録**: 日付、タイトル、内容を入力して日報を作成
* **日報更新**: 登録済み日報の編集
* **日報削除**: 日報情報の論理削除

## データベース・テーブル構成
従業員テーブルと日報テーブル間で「1対多」のリレーションを構築しています。
また、データの削除については物理削除ではなく、削除フラグ（`delete_flg`）を用いた**論理削除**を採用しています。

* `employees` (従業員テーブル)
* `reports` (日報テーブル)
